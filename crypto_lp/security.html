🔐 情報セキュリティ方針

最終更新日：2026年1月5日

株式会社STeLA（以下「当社」といいます）は、お客様の大切な資産と情報を守ることを最優先の使命と考えています。

NEO Systemをご利用いただく全てのお客様に安心してサービスをご利用いただくため、以下の情報セキュリティ方針を定め、これを遵守いたします。

1. 基本方針

当社は、仮想通貨自動運用システム「NEO System」の提供にあたり、お客様からお預かりする個人情報、取引情報、資産情報等（以下「情報資産」といいます）の機密性、完全性、可用性を確保するため、組織的、技術的、物理的な安全管理措置を講じます。

2. 情報セキュリティ管理体制

2.1 組織体制

  • 情報セキュリティ責任者の設置：代表取締役を最高情報セキュリティ責任者（CISO）として任命し、情報セキュリティに関する統括的な責任を負います
  • セキュリティチームの編成：専門的な知識を有するセキュリティエンジニアを中心としたチームを編成し、24時間365日の監視体制を構築しています
  • 定期的な会議開催：月次でセキュリティ会議を開催し、脅威情報の共有、対策の検討、インシデントの振り返りを実施します

2.2 規程・手順の整備

  • 情報セキュリティ管理規程
  • 個人情報取扱規程
  • アクセス制御規程
  • インシデント対応手順書
  • システム開発・運用セキュリティ基準
  • 外部委託管理規程

3. 技術的セキュリティ対策

3.1 暗号化技術

  • 通信の暗号化：全ての通信にTLS 1.3以上の暗号化プロトコルを採用し、通信内容の盗聴・改ざんを防止します
  • データベース暗号化：お客様の個人情報、取引履歴等は全てAES-256による暗号化を施して保管します
  • 秘密鍵の管理：暗号化に使用する秘密鍵は、ハードウェアセキュリティモジュール（HSM）で厳重に管理します

3.2 アクセス制御

  • 多要素認証（MFA）：管理画面へのアクセスには、パスワードに加えてワンタイムパスワード等の多要素認証を必須としています
  • 最小権限の原則：従業員には業務上必要最小限のアクセス権限のみを付与し、定期的に権限の見直しを実施します
  • アクセスログの記録：全てのシステムアクセスをログに記録し、不正アクセスの検知と追跡を可能にします

3.3 ウォレットセキュリティ

  • マルチシグネチャウォレット：お客様の資産は複数の署名を必要とするマルチシグウォレットで管理し、単一障害点を排除します
  • コールドウォレット保管：大部分の資産はオフライン環境（コールドウォレット）で保管し、ハッキングリスクを最小化します
  • ホットウォレットの制限：日々の運用に必要な最小限の資産のみをホットウォレットで管理します
  • 出金承認プロセス：一定額以上の出金には複数の担当者による承認を必須とする多段階承認プロセスを実装しています

3.4 ネットワークセキュリティ

  • ファイアウォール：次世代ファイアウォール（NGFW）により、不正な通信を遮断します
  • 侵入検知・防御システム（IDS/IPS）：24時間365日、ネットワークトラフィックを監視し、異常を検知します
  • DDoS対策：分散型サービス拒否攻撃に対する多層防御を実装し、サービスの可用性を確保します
  • ネットワーク分離：本番環境、開発環境、管理ネットワークを物理的・論理的に分離します

3.5 脆弱性管理

  • 定期的な脆弱性診断：外部の専門機関による脆弱性診断を四半期に1回実施します
  • ペネトレーションテスト：年1回、実際の攻撃を想定した侵入テストを実施し、防御体制の有効性を検証します
  • パッチ管理：OSやミドルウェア、アプリケーションの脆弱性情報を常時監視し、重要度に応じて迅速にパッチを適用します

4. 物理的セキュリティ対策

4.1 データセンターセキュリティ

  • Tier3以上のデータセンター：国内外の高セキュリティデータセンターを利用し、物理的な安全性を確保します
  • 入退室管理：生体認証と多要素認証による厳格な入退室管理を実施します
  • 24時間監視体制：監視カメラと警備員による24時間365日の監視体制を維持します
  • 環境管理：温度、湿度、電源供給を厳格に管理し、機器の安定稼働を保証します

4.2 オフィスセキュリティ

  • ICカードによる入退室管理
  • 来訪者の記録と同行ルールの徹底
  • クリアデスクポリシーの実施（机上に機密情報を放置しない）
  • シュレッダーによる文書廃棄の徹底

5. 人的セキュリティ対策

5.1 従業員教育

  • 入社時研修：全従業員に対して情報セキュリティに関する基礎教育を実施します
  • 定期研修：年2回、最新の脅威情報や対策に関する研修を実施します
  • フィッシング訓練：定期的に模擬フィッシングメールを送信し、従業員のセキュリティ意識を高めます
  • 専門研修：技術者向けに、セキュアコーディング、脆弱性対策等の専門的な研修を実施します

5.2 契約・誓約

  • 全従業員と秘密保持契約を締結
  • 情報セキュリティ誓約書の提出
  • 退職時の情報返却・削除の徹底

6. インシデント対応

6.1 監視・検知体制

  • SIEM（Security Information and Event Management）：各種ログを統合的に分析し、異常を早期に検知します
  • SOC（Security Operation Center）：セキュリティ専門チームが24時間365日体制で監視します
  • 自動アラート：異常検知時には即座に担当者に通知が届く仕組みを構築しています

6.2 対応手順

  1. 検知：自動監視システムまたは担当者がインシデントを検知
  2. 初動対応：被害拡大防止のため、該当システムの隔離等の緊急措置を実施
  3. 調査・分析：インシデントの原因、影響範囲を詳細に調査
  4. 復旧：システムの復旧と正常性の確認
  5. 報告：必要に応じて、お客様、監督官庁への報告
  6. 再発防止：原因分析に基づく再発防止策の策定と実施

7. 外部委託管理

当社は、業務の一部を外部に委託する場合、以下の管理を実施します。

  • 委託先の適格性評価（セキュリティ体制、実績等の確認）
  • 秘密保持契約の締結
  • 委託契約書へのセキュリティ要件の明記
  • 定期的な監査の実施
  • 再委託の制限と事前承認制

8. 第三者認証・監査

当社は、情報セキュリティ体制の客観性と信頼性を確保するため、以下の取り組みを行っています。

  • ISO27001認証取得準備：国際標準の情報セキュリティマネジメントシステム（ISMS）の認証取得を目指しています
  • 外部監査の受入：年1回、外部の専門監査機関による監査を受け、改善点の洗い出しと対策を実施します
  • 第三者セキュリティ評価：スマートコントラクトやシステムアーキテクチャについて、定期的に第三者評価を受けています

9. 継続的改善

当社は、情報セキュリティを継続的に改善するため、以下のサイクルを実施します。

  1. Plan（計画）：リスク評価に基づくセキュリティ計画の策定
  2. Do（実施）：計画に基づく対策の実施と運用
  3. Check（評価）：内部監査、外部監査、ペネトレーションテスト等による有効性の評価
  4. Act（改善）：評価結果に基づく改善措置の実施

10. お客様へのお願い

当社は最大限のセキュリティ対策を講じておりますが、お客様ご自身でも以下の点にご注意ください。

  • パスワードは複雑で推測されにくいものを設定し、定期的に変更してください
  • パスワードは他のサービスと使い回さないでください
  • フィッシングメールや不審なリンクに注意してください
  • 公共のWi-Fiなど、セキュリティが確保されていないネットワークでのアクセスは避けてください
  • ログイン情報を他人と共有しないでください
  • 不審なアクセスに気付いた場合は、直ちに当社までご連絡ください
